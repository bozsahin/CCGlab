CCGlab
Lisp code for the masses, for something not so massive.

CCGlab is a tool to experiment with Combinatory Categorial Grammar (CCG).

It is designed with linguists, cognitive scientists, and computational linguists in mind. It can be used for quick check of analyses, or for large-scale development of grammars and models.

CCGlab grammars are written much like in CCG papers,The output is almost in papers format (well, almost).

It implements all established combinators of CCG, also, experimental ones, in all variants and powers. Unary rules, slash modalities, and meta-categories such as (X\X)/X have been implemented. The manual describes the range of possible CCGlab grammars.

It also implements probabilistic CCG, aka. PCCG. In the plain vanilla version, only lexical features are used, which you can amend. Parameter estimation and update are provided with or without extrapolation. Therefore model training is possible. Models and grammars use the same format.

In deduction mode, CCGlab gives you all parses of a string, along with logical forms. In induction mode, it gives you the most likely LF for a string, its most likely derivation, and the most likely derivation for any LF for the string. Both are based on the same CKY parse engine and the same grammar format.

The main features of CCGlab which distinguish it from other publicly available CCG code:

  • The logical form is associated with the entire category, via self-contained and lisp-free lambda-calculus.
  • Structural unification plays no linguistic role; basic categories and their atomic features are matched for term consistency only.
  • By default standard combinators are available all the time, in their powers (linear, quadratic, cubic). They all have on/off switches for experimentation.
  • Type-raising is not built-in, because it does not combine. Facilities are provided for it, such as lexical rules and morpholexical generalizations. There is a compiler for it.
  • We can do deduction on a statistically-trained grammar, and induction on a linguistically-motivated grammar, because the grammar format is the same, and feature counts are always calculated by CKY independent of mode of use. The first one is helpful in fine-tuning a model. The second one is helpful in prioritizing the lexical assumptions.
